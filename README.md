# Location-Recording-App
💡 [ Protfolio Project 008] 위치 기록 앱 제작 프로젝트 

## 📌 프로젝트 소개
이 프로젝트는 사용자의 위치를 백그라운드에서 주기적으로 추적하여 데이터베이스에 기록하고, 구글 맵을 통해 이동 경로를 시각화하는 안드로이드 앱입니다. 사용자는 캘린더를 통해 특정 날짜의 이동 기록을 조회할 수 있으며, 전체 경로 보기와 경로 검색 기능을 통해 위치 정보를 손쉽게 확인할 수 있습니다. 위치 데이터는 SQLite를 이용해 로컬에 저장되며, 원격 서버 MariaDB를 통해 데이터를 백업 및 관리할 수 있습니다.

## 📌 주요 기능
### - 사용자 로그인 및 인증:
사용자는 서버에 저장된 자격 정보를 통해 로그인할 수 있으며, 웹 서버와 PHP 스크립트를 사용해 인증을 처리합니다.  

### - 위치 추적 및 기록:
- 앱은 백그라운드에서 1분마다 사용자의 위치를 추적하여 데이터를 기록합니다.  
- 50미터 이상 이동 시 새로운 위치 정보가 기록되며, 같은 위치에 10분 이상 머물면 해당 정보를 데이터베이스에 저장합니다.  

### - 캘린더 기반 이동 기록 조회:
사용자는 캘린더에서 날짜를 선택해 그날의 이동 기록을 조회할 수 있으며, 해당 기록을 클릭하면 구글 맵에 경로가 표시됩니다.  

### - 구글 맵 경로 시각화:
사용자의 특정 날짜 또는 전체 경로를 구글 맵을 통해 시각적으로 확인할 수 있으며, 각 위치에 마커가 표시되어 머문 시간과 장소를 확인할 수 있습니다.  

### - 검색 기능:
특정 날짜 또는 장소 이름으로 경로 및 위치 기록을 검색할 수 있습니다.  

### - 데이터베이스 관리 및 동기화:
- 앱은 SQLite를 이용한 로컬 데이터베이스를 통해 위치 기록을 관리하고, MariaDB를 이용해 원격 서버에 데이터를 백업합니다.  
- 데이터는 날짜별 또는 장소 이름으로 검색할 수 있으며, 데이터의 삽입, 수정, 삭제가 가능합니다.  

## 📌 구현 상세
### - MainActivity.java:
메인 화면을 구성하며, 캘린더 뷰와 리스트 뷰를 통해 이동 기록을 조회할 수 있습니다. 사용자가 날짜를 선택하면 해당 날짜의 이동 기록이 리스트에 표시되고, 기록을 클릭하면 구글 맵에 해당 경로가 표시됩니다. 또한, 백그라운드 위치 추적 서비스를 시작합니다.  

### - LocationService.java:
사용자의 위치를 백그라운드에서 추적하며, 일정 시간 동안 같은 위치에 머물거나 50미터 이상 이동했는지 확인한 후, 해당 데이터를 SQLite 데이터베이스에 기록합니다. 해당 데이터는 나중에 원격 서버로 동기화됩니다.  

### - RouteView.java:
구글 맵을 사용해 사용자의 특정 날짜의 경로를 시각적으로 표시하며, 경로를 따라 마커를 배치하여 해당 위치에 머문 시간과 장소를 보여줍니다.  

### - SearchView.java:
특정 날짜 또는 장소 이름으로 위치 기록을 검색하여 관련 데이터를 조회하는 기능을 담당합니다. 검색된 기록을 리스트뷰로 보여주고 클릭 시 경로를 확인할 수 있습니다.  

### - Constants.java:
서비스 시작 및 종료에 관련된 상수를 정의하며, 앱 내에서 자주 사용하는 고정 값들을 관리합니다.
